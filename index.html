A Topological Proof That the Surface Code Has Distance $L$

A mnemonic medium essay — prompts after each section help you retain the key ideas

Why this post?

If you've read an introduction to the toric code — say, Arthur Pesah's excellent interactive post — you've probably seen the claim that the toric code on an $L \times L$ lattice has distance $L$. And you probably feel like you understand why: the shortest non-contractible loop on the torus has $L$ edges, so the lightest non-trivial logical operator has weight $L$.

It's worth examining this argument carefully, because one of its steps is doing more work than it might seem.

The standard argument and its gap

The argument, as typically presented, has three steps:

  1. Logical operators are loops. An X-type operator on some edge-set $c$ commutes with all Z-stabilizers if and only if $c$ forms a cycle (every vertex has even degree). It acts trivially on the codespace (i.e., it is a stabilizer) if and only if $c$ is a boundary — the perimeter of some set of faces. So non-trivial logical operators correspond to cycles that are not boundaries: non-contractible loops.
  2. There exist non-contractible loops of length $L$. A horizontal row of $L$ edges wrapping around the torus is such a loop. So the distance is at most $L$.
  3. No non-contractible loop has length less than $L$. A non-contractible loop must "wrap all the way around" the torus, which requires at least $L$ edges. Therefore the distance is at least $L$.

Steps 1 and 2 are solid. Step 3 is the problem. It feels obvious — of course you need at least $L$ edges to get around an $L \times L$ torus. But "wrapping all the way around" is doing a lot of informal work here. A non-contractible loop doesn't have to look like a straight horizontal line. It could zig-zag, include vertical segments, wander wildly across the lattice. Why can't some clever squiggly path wrap around the torus using fewer than $L$ edges? The geometric intuition is compelling, but it isn't a proof.

To make it a proof, we need to answer two questions precisely:

  • What exactly distinguishes a non-contractible loop from a contractible one? We need a formal invariant — a quantity we can compute from a cycle that is zero for all boundaries and nonzero for non-contractible loops.
  • Why does this invariant force a minimum weight? We need to show that any cycle with a nonzero invariant must use at least $L$ edges.

The topological approach

This is where the language of homology enters. The key idea is to define two $\mathbb{F}_2$-valued invariants, $h(c)$ and $v(c)$, that detect whether a cycle "wraps around" the torus horizontally or vertically. These invariants have three crucial properties:

  • They are well-defined on homology classes: adding a boundary to a cycle doesn't change its $h$ or $v$ values.
  • They completely classify the four cosets of the first homology group $H_1$. A cycle is a boundary if and only if $h = v = 0$.
  • They force a minimum weight. If $h(c) = 1$, then $c$ must contain at least one horizontal edge at each of $L$ distinct $x$-coordinates — giving at least $L$ edges. Similarly for $v(c) = 1$.

What you'll need

This post assumes familiarity with the stabilizer formalism (stabilizer groups, codespace, logical operators, distance). If you need a refresher, Arthur Pesah's series on the stabilizer formalism is a great starting point. We'll also use basic linear algebra over $\mathbb{F}_2$ (vector spaces, kernels, rank-nullity), but no prior knowledge of algebraic topology — we build up everything we need from scratch.

Roadmap

Sections 1–3 set up the lattice, define the chain complex, and introduce cycles, boundaries, and the first homology group $H_1$. Section 4 connects this formalism to the toric code. Section 5 computes $\dim(H_1) = 2$ using rank-nullity. Section 6 defines the invariants $h$ and $v$. Section 7 proves that these invariants give an isomorphism $H_1 \cong \mathbb{F}_2^2$. Section 8 combines everything to prove that the distance is exactly $L$.

1. The Lattice

We work with the surface code on a torus (periodic boundary conditions), sometimes called the toric code. The lattice is an $L \times L$ square grid, with all arithmetic on coordinates taken modulo $L$.

  • Vertices: $(x, y)$ for $x, y \in \{0, 1, \ldots, L-1\}$. There are $L^2$ vertices.
  • Edges: Two types.
    • Horizontal edges: $H(x, y)$ connects $(x, y)$ to $(x+1, y)$.
    • Vertical edges: $V(x, y)$ connects $(x, y)$ to $(x, y+1)$.
    • In total, $n = 2L^2$ edges.
  • Faces: $F(x, y)$ is the unit square with corners $(x,y)$, $(x+1,y)$, $(x,y+1)$, $(x+1,y+1)$. There are $L^2$ faces, with boundary edges Bottom $H(x,y)$, Top $H(x,y+1)$, Left $V(x,y)$, Right $V(x+1,y)$.

Each edge borders exactly two faces. Each vertex is incident to exactly four edges: $H(x-1,y)$, $H(x,y)$, $V(x,y-1)$, $V(x,y)$.

The toric code places one qubit on each of the $2L^2$ edges and defines:

  • X-type (plaquette) stabilizers: For each face $f$, apply $X$ to the four boundary edges of $f$.
  • Z-type (vertex) stabilizers: For each vertex $v$, apply $Z$ to the four edges incident to $v$.
Review — The Lattice
In the toric code on an $L \times L$ lattice, where are the qubits placed?
On the edges of the lattice. There are $2L^2$ qubits total ($L^2$ horizontal edges and $L^2$ vertical edges).

X-type (plaquette) stabilizers, which apply $X$ to the four edges bounding a face, and Z-type (vertex) stabilizers, which apply $Z$ to the four edges incident to a vertex.

Exactly two, because of the periodic boundary conditions. (No edge is on a "boundary" of the lattice.)

2. The Chain Complex over $\mathbb{F}_2$

Our goal is to reason algebraically about subsets of edges. We represent edge-sets, vertex-sets, and face-sets as vectors over $\mathbb{F}_2 = \{0,1\}$, and the geometric notion of "taking the boundary" becomes a linear map.

Definition ($k$-cells and chains)

The $k$-cells of the lattice are:

  • 0-cells (vertices)
  • 1-cells (edges)
  • 2-cells (faces)

A $k$-chain is an $\mathbb{F}_2$-linear combination of $k$-cells — equivalently, a subset of $k$-cells, where addition is symmetric difference. The chain spaces are:

  • $C_0 = \mathbb{F}_2^{L^2}$ — subsets of vertices
  • $C_1 = \mathbb{F}_2^{2L^2}$ — subsets of edges
  • $C_2 = \mathbb{F}_2^{L^2}$ — subsets of faces

The weight $|c|$ of a 1-chain is the number of edges with coefficient 1.

Definition (Boundary maps)
$\partial_2 : C_2 \to C_1$ sends a face to the sum of its four boundary edges: $$\partial_2(F(x,y)) = H(x,y) + H(x,y+1) + V(x,y) + V(x+1,y)$$ An edge $e$ is in $\partial_2(\mathcal{F})$ iff exactly one of the two faces bordering $e$ is in $\mathcal{F}$. $\partial_1 : C_1 \to C_0$ sends an edge to the sum of its two endpoints: $\partial_1(H(x,y)) = (x,y)+(x+1,y)$, $\quad\partial_1(V(x,y)) = (x,y)+(x,y+1)$. A vertex $v$ is in $\partial_1(E)$ iff an odd number of edges in $E$ are incident to $v$.
Lemma 2.1
$\partial_1 \circ \partial_2 = 0$.
Proof. Apply both maps to a single face $F(x,y)$: $$\partial_1(\partial_2(F(x,y))) = [(x,y)+(x+1,y)] + [(x,y+1)+(x+1,y+1)] + [(x,y)+(x,y+1)] + [(x+1,y)+(x+1,y+1)]$$ Each of the four corner vertices appears exactly twice, so each cancels over $\mathbb{F}_2$. $\square$
Review — The Chain Complex
What does the boundary map $\partial_2$ do to a single face $F(x,y)$?
It returns the sum (over $\mathbb{F}_2$) of the four edges bounding that face: $H(x,y) + H(x,y+1) + V(x,y) + V(x+1,y)$.
Recalled?
For a set of faces $\mathcal{F}$, when is an edge $e$ in $\partial_2(\mathcal{F})$?
When exactly one of the two faces bordering $e$ is in $\mathcal{F}$.
Recalled?
What does the property $\partial_1 \circ \partial_2 = 0$ mean geometrically?
The boundary of a boundary is empty — the perimeter of any set of faces has no dangling endpoints (every vertex appears an even number of times).

3. Cycles, Boundaries, and Homology

A cycle is an edge-set with no dangling ends (every vertex has even degree). A boundary is the perimeter of some set of faces. Every boundary is automatically a cycle, but some cycles wrap around the torus without being any boundary — these are the non-trivial logical operators.

Definition
  • $Z_1 = \ker(\partial_1) \subseteq C_1$: the space of 1-cycles.
  • $B_1 = \operatorname{im}(\partial_2) \subseteq C_1$: the space of 1-boundaries.
Lemma 3.1
$B_1 \subseteq Z_1$. Every boundary is a cycle.
Proof. If $c = \partial_2(\mathcal{F})$, then $\partial_1(c) = \partial_1(\partial_2(\mathcal{F})) = 0$ by Lemma 2.1. $\square$
Definition (First homology group)
$$H_1 = Z_1 / B_1$$ Two cycles $c, c'$ are homologous ($c \sim c'$) if $c + c' \in B_1$. They represent the same coset of $H_1$.
Review — Cycles, Boundaries, Homology
What is a 1-cycle?
An edge-set where every vertex has even degree — 0, 2, or 4 edges of the set meet at each vertex.
Recalled?
What is a 1-boundary?
An edge-set that equals $\partial_2(\mathcal{F})$ for some set of faces $\mathcal{F}$ — the perimeter of a set of faces.
Recalled?
Why is every boundary automatically a cycle?
Because $\partial_1 \circ \partial_2 = 0$: applying $\partial_1$ to any boundary gives 0, meaning every vertex has even degree.
Recalled?
When are two cycles $c$ and $c'$ homologous?
When they differ by a boundary: $c + c' \in B_1$. Equivalently, they represent the same element of $H_1 = Z_1 / B_1$.

4. Connection to the Toric Code

Non-trivial X-type logicals correspond exactly to non-trivial elements of $H_1$ (cycles that aren't boundaries). The distance question becomes: what is the lightest non-trivial cycle?

4.1 X-type logicals as non-trivial cycles

Claim 4.1
An X-type operator on edge-set $c$ commutes with all Z-type stabilizers if and only if $c \in Z_1$.
Proof. The vertex stabilizer at $v$ applies $Z$ to the four edges incident to $v$. The X-type operator on $c$ anticommutes on each qubit where both operators act; the total sign is $(-1)^{|c \cap \operatorname{supp}(v)|}$. This is $+1$ for all $v$ iff every vertex has even degree in $c$, i.e., $c \in Z_1$. $\square$
Claim 4.2
An X-type operator on edge-set $c$ is a product of plaquette stabilizers if and only if $c \in B_1$.
Proof. A product of plaquette stabilizers for face-set $\mathcal{F}$ is the X-type operator on $\partial_2(\mathcal{F})$. Every element of $B_1 = \operatorname{im}(\partial_2)$ arises this way. $\square$

Combining: an X-type operator on $c$ is a non-trivial X-type logical iff $c \in Z_1 \setminus B_1$, i.e., $c$ represents a non-trivial element of $H_1$. The X-distance is: $$d_X = \min\{|c| : c \in Z_1 \setminus B_1\}$$

4.2 Z-type logicals and the dual lattice

Definition (Dual lattice)
The dual lattice is obtained by placing a dual vertex $f^*$ at the center of each face $f$, a dual edge $e^*$ for each primal edge $e$ (connecting the two adjacent dual vertices), and a dual face $v^*$ for each primal vertex $v$. The dual is again an $L \times L$ torus with the same combinatorial structure.

A Z-type operator on primal edge-set $c$ commutes with all X-type stabilizers iff the dual edge-set $c^* = \{e^* : e \in c\}$ is a cycle in the dual lattice. It is a product of vertex stabilizers iff $c^*$ is a boundary. So $d_Z$ = minimum weight of a non-trivial cycle in the dual lattice. We focus on $d_X = L$ first (Sections 5–8.2).

Review — Connection to the Toric Code

$c$ is a cycle — every vertex has even degree in $c$.
Recalled?
A non-trivial X-type logical operator corresponds to what kind of cycle?
A cycle that is not a boundary — a non-trivial element of $H_1 = Z_1 / B_1$.
Recalled?
What is the dual lattice of the $L \times L$ torus, and why is it useful?
It places a vertex at the center of each primal face, connects them by dual edges (one per primal edge), and turns primal vertices into dual faces. Z-type logicals on the primal lattice correspond to cycles on the dual lattice, so we can reuse the same homological framework for both X-type and Z-type distance.

5. Dimension of $H_1$

We prove $\dim(H_1) = 2$, which tells us the toric code encodes $k = 2$ logical qubits and $H_1$ has $2^2 = 4$ cosets. Since $H_1 = Z_1/B_1$, we need $\dim(Z_1)$ and $\dim(B_1)$ separately. Our tool is rank-nullity.

Theorem (Rank-nullity)
For a linear map $T: V \to W$: $\;\dim(V) = \dim(\ker T) + \operatorname{rank}(T)$. Also, $\operatorname{rank}(T) = \operatorname{rank}(T^T)$ (rank equals rank of transpose over any field).

5.2 Computing $\dim(Z_1)$

$Z_1 = \ker(\partial_1)$. By rank-nullity: $2L^2 = \dim(Z_1) + \operatorname{rank}(\partial_1)$. We compute $\operatorname{rank}(\partial_1)$ via its transpose.

$\partial_1^T: C_0 \to C_1$ maps a vertex-subset $S$ to the edge cut of $S$: edges with exactly one endpoint in $S$.

Lemma 5.1
$\dim(\ker(\partial_1^T)) = 1$.
Proof. $S \in \ker(\partial_1^T)$ iff no edge has exactly one endpoint in $S$, i.e., every edge has both or neither endpoints in $S$. Since the lattice is connected, $S$ must be $\emptyset$ or all vertices. $\square$
Corollary 5.2
$\operatorname{rank}(\partial_1) = L^2 - 1$.
Proof. By rank-nullity on $\partial_1^T: C_0 \to C_1$: $\operatorname{rank}(\partial_1^T) = L^2 - 1$. Since $\operatorname{rank}(\partial_1) = \operatorname{rank}(\partial_1^T)$, done. $\square$
Corollary 5.3
$\dim(Z_1) = L^2 + 1$.
Proof. $\dim(Z_1) = 2L^2 - (L^2-1) = L^2+1$. $\square$

5.3 Computing $\dim(B_1)$

$B_1 = \operatorname{im}(\partial_2)$, so $\dim(B_1) = \operatorname{rank}(\partial_2)$. By rank-nullity: $L^2 = \dim(\ker(\partial_2)) + \dim(B_1)$.

Lemma 5.4
$\dim(\ker(\partial_2)) = 1$.
Proof. $\mathcal{F} \in \ker(\partial_2)$ iff every edge borders an even number of faces in $\mathcal{F}$. Since each edge borders exactly two faces, adjacent faces must both be in $\mathcal{F}$ or both absent. By connectedness of the face-adjacency graph, $\mathcal{F}$ is $\emptyset$ or all faces. $\square$
Corollary 5.5
$\dim(B_1) = L^2 - 1$.
Proof. $\dim(B_1) = L^2 - 1$. $\square$

5.4 The dimension of $H_1$

Theorem 5.6
$\dim(H_1) = 2$.
Proof. $$\dim(H_1) = \dim(Z_1) - \dim(B_1) = (L^2+1) - (L^2-1) = 2. \quad\square$$

Since $H_1$ is a 2-dimensional vector space over $\mathbb{F}_2$, it has exactly $2^2 = 4$ elements (cosets).

Review — Dimension of $H_1$
What is $\dim(H_1)$ for the $L \times L$ torus over $\mathbb{F}_2$?
2. This means the toric code encodes 2 logical qubits and $H_1$ has $2^2 = 4$ cosets.
Recalled?
In the computation of $\dim(Z_1)$, why do we analyze the transpose $\partial_1^T$ instead of $\partial_1$ directly?
Because $\ker(\partial_1^T)$ is easy to characterize: it consists of vertex-labelings where adjacent vertices always agree, which forces the labeling to be constant (by connectedness). This gives $\dim(\ker(\partial_1^T)) = 1$, so $\operatorname{rank}(\partial_1) = L^2 - 1$.
Recalled?
Why is $\dim(\ker(\partial_2)) = 1$?
A face-set $\mathcal{F}$ is in $\ker(\partial_2)$ iff every edge borders an even number of faces in $\mathcal{F}$. Since each edge borders exactly two faces, adjacent faces must have the same membership. Connectedness forces $\mathcal{F}$ to be either empty or all faces.

6. The Invariants $h$ and $v$

We know $H_1 \cong \mathbb{F}_2^2$ as sets (4 cosets), but how do we tell which coset a cycle belongs to? We define two parity-valued invariants.

6.1 Definitions

Definition
For $c \in C_1$ and $x_0 \in \{0,\ldots,L-1\}$: $$h_{x_0}(c) = \bigl|\{y : H(x_0, y) \in c\}\bigr| \bmod 2$$ the parity of horizontal edges of $c$ at $x$-coordinate $x_0$. Similarly: $$v_{y_0}(c) = \bigl|\{x : V(x, y_0) \in c\}\bigr| \bmod 2$$ the parity of vertical edges of $c$ at $y$-coordinate $y_0$.
Lemma 6.1
The maps $h_{x_0}, v_{y_0}: C_1 \to \mathbb{F}_2$ are linear.
Proof. Over $\mathbb{F}_2$, the indicator of "exactly one of $c, c'$ contains edge $e$" equals the sum of the two indicators. So $h_{x_0}(c + c') = h_{x_0}(c) + h_{x_0}(c')$. $\square$

6.2 Independence of coordinate choice

Lemma 6.2
If $c \in Z_1$, then $h_{x_0}(c)$ is the same for all $x_0$, and $v_{y_0}(c)$ is the same for all $y_0$.
Proof. Sum the even-degree condition over the $L$ vertices in column $x = x_0 + 1$: $$\sum_y \deg_c(x_0+1,y) \equiv 0 \pmod{2}$$ Expanding: edges incident to each $(x_0+1,y)$ are $H(x_0,y)$, $H(x_0+1,y)$, $V(x_0+1,y-1)$, $V(x_0+1,y)$. Summing over all $y$, the two vertical sums run over the same $L$ edges (indices are mod $L$) and cancel: $$\sum_y [H(x_0,y) \in c] + \sum_y [H(x_0{+}1,y) \in c] \equiv 0 \pmod 2$$ So $h_{x_0}(c) = h_{x_0+1}(c)$. $\square$

For cycles we write simply $h(c)$ and $v(c)$ since the values are coordinate-independent.

6.3 Invariance under adding boundaries

Lemma 6.3
If $b \in B_1$, then $h(b) = 0$ and $v(b) = 0$.
Proof. Let $b = \partial_2(\mathcal{F})$. Write $a_y = [F(x_0,y) \in \mathcal{F}]$. Then $[H(x_0,y) \in b] = a_y \oplus a_{y-1}$. Summing over $y$: $$h_{x_0}(b) = \sum_y (a_y \oplus a_{y-1}) = \sum_y a_y + \sum_y a_{y-1} = 2\sum_y a_y \equiv 0 \pmod 2$$ since both sums run over the same values (indices mod $L$). $\square$
Corollary 6.4 & Lemma 6.5
The map $\varphi: H_1 \to \mathbb{F}_2^2$ defined by $\varphi([c]) = (h(c), v(c))$ is well-defined and linear.
Proof. Well-defined: if $c' = c + b$ with $b \in B_1$, then $h(c') = h(c) + h(b) = h(c)$ by Lemmas 6.1 and 6.3. Linear: $\varphi([c_1]+[c_2]) = (h(c_1+c_2), v(c_1+c_2)) = (h(c_1)+h(c_2), v(c_1)+v(c_2)) = \varphi([c_1])+\varphi([c_2])$. $\square$
Review — The Invariants $h$ and $v$
What does $h(c)$ measure?
The parity (mod 2) of the number of horizontal edges of $c$ at any fixed $x$-coordinate.
Recalled?
Why is $h(c)$ the same at every $x$-coordinate (for a cycle $c$)?
Summing the even-degree condition over a column of vertices at $x = x_0 + 1$: the two vertical edge sums cancel (they run over the same set mod $L$), leaving $h_{x_0}(c) + h_{x_0+1}(c) \equiv 0$.

Both are 0. This is why $\varphi([c]) = (h(c), v(c))$ is well-defined on homology classes: adding a boundary doesn't change $h$ or $v$.

7. The Isomorphism $H_1 \cong \mathbb{F}_2^2$

We now show $\varphi: H_1 \to \mathbb{F}_2^2$ is an isomorphism, meaning $h$ and $v$ completely classify cosets: two cycles are homologous iff they have the same $(h,v)$ values. This guarantees every non-trivial cycle has $h=1$ or $v=1$ (or both) — the key to the distance lower bound.

Theorem 7.1
$\varphi: H_1 \to \mathbb{F}_2^2$ is an isomorphism.
Proof. $\dim(H_1) = 2 = \dim(\mathbb{F}_2^2)$, so it suffices to show surjectivity.

Representative for $(1,0)$: Let $c_h = \{H(x,0) : x=0,\ldots,L-1\}$. This is a cycle (each vertex meets 0 or 2 edges of $c_h$), has $h(c_h) = 1$ (exactly one horizontal edge per $x$-slice), $v(c_h)=0$ (no vertical edges), and is not a boundary since $h(c_h)=1 \neq 0$ while all boundaries have $h=0$. So $\varphi([c_h]) = (1,0)$.

Representative for $(0,1)$: $c_v = \{V(0,y): y=0,\ldots,L-1\}$. By the symmetric argument, $\varphi([c_v]) = (0,1)$.

Representative for $(1,1)$: $c_{hv} = c_h + c_v$. By linearity of $\varphi$: $\varphi([c_{hv}]) = (1,0)+(0,1) = (1,1)$.

Thus $\varphi$ is surjective, hence an isomorphism. $\square$
Review — The Isomorphism $H_1 \cong \mathbb{F}_2^2$
What are the four cosets of $H_1$, and what $(h, v)$ values do they have?
$(0,0)$: boundaries (trivial). $(1,0)$: horizontal loops. $(0,1)$: vertical loops. $(1,1)$: both.
Recalled?
How do we know the horizontal loop $c_h$ is not a boundary?
$h(c_h) = 1$, but every boundary has $h = 0$ (Lemma 6.3). So $c_h$ cannot be a boundary.
Recalled?
Why does surjectivity of $\varphi: H_1 \to \mathbb{F}_2^2$ imply that $\varphi$ is an isomorphism?
$\varphi$ is a linear map between vector spaces of the same dimension (both 2). A surjective linear map between equal-dimensional spaces is automatically injective.

8. The Distance Theorem

We now have all the tools. The isomorphism $\varphi: H_1 \to \mathbb{F}_2^2$ tells us every non-trivial cycle has $h(c)=1$ or $v(c)=1$ (or both). Lemma 6.2 says this parity is the same across all $L$ slices. Together, these force every non-trivial cycle to contribute at least one edge to each of $L$ disjoint sets.

Theorem 8.1
The X-distance of the toric code on an $L \times L$ lattice is $d_X = L$.

8.1 Upper bound: $d_X \leq L$

The horizontal loop $c_h = \{H(x,0): x=0,\ldots,L-1\}$ is non-trivial with $|c_h| = L$, so $d_X \leq L$.

8.2 Lower bound: $d_X \geq L$

Let $c \in Z_1 \setminus B_1$. Since $\varphi$ is injective and $c \notin B_1$, we have $\varphi([c]) \neq (0,0)$, so $h(c)=1$ or $v(c)=1$.

Case 1: $h(c) = 1$. Define $S_x = \{H(x,y): y = 0,\ldots,L-1\}$ for each $x$. These $L$ sets are pairwise disjoint. By Lemma 6.2, the parity of $|c \cap S_x|$ equals $h(c) = 1$ for every $x$, so $|c \cap S_x| \geq 1$ for each $x$. Therefore: $$|c| \geq \sum_{x=0}^{L-1} |c \cap S_x| \geq L$$

Case 2: $v(c) = 1$. The same argument with $T_y = \{V(x,y): x=0,\ldots,L-1\}$ gives $|c| \geq L$. $\square$

8.3 Z-type logicals and the full distance

Claim 8.2
For a CSS code, $d = \min(d_X, d_Z)$.
Proof.

Any Pauli $P$ can be written as $P = X^a Z^b$ with $a,b \in \mathbb{F}_2^n$. In a CSS code, $P$ commutes with all stabilizers iff $a \in Z_1$ and $b \in Z_1^*$ independently, and $P$ is a stabilizer iff $a \in B_1$ and $b \in B_1^*$. So $P$ is a non-trivial logical iff it commutes with all stabilizers and $a \in Z_1\setminus B_1$ or $b \in Z_1^*\setminus B_1^*$.

Lower bound: if $a \in Z_1\setminus B_1$ then $\operatorname{wt}(P) \geq |a| \geq d_X$; if $b \in Z_1^*\setminus B_1^*$ then $\operatorname{wt}(P) \geq |b| \geq d_Z$. Since at least one holds, $\operatorname{wt}(P) \geq \min(d_X,d_Z)$.

Upper bound: a minimum-weight non-trivial X-type logical has weight $d_X$, so $d \leq d_X$; similarly $d \leq d_Z$. Thus $d = \min(d_X, d_Z)$. $\square$

The dual lattice has the same structure as the primal, so the entire argument gives $d_Z = L$ as well. Therefore:

$$d = \min(d_X, d_Z) = \min(L, L) = L$$

The toric code is a $[[2L^2, 2, L]]$ code: $n = 2L^2$ physical qubits, $k = 2$ logical qubits, $d = L$. $\quad\blacksquare$

Review — The Distance Theorem
What is the upper bound argument for $d_X \leq L$?
The horizontal loop $c_h$ has exactly $L$ edges and is a non-trivial cycle (since $h(c_h) = 1 \neq 0$). So $d_X \leq L$.
Recalled?
Why does $h(c) = 1$ imply that $|c| \geq L$?
$h(c) = 1$ means $c$ has an odd number of horizontal edges at each of the $L$ distinct $x$-coordinates. Odd $\geq 1$, and these $L$ sets are pairwise disjoint, so $|c| \geq L$.
Recalled?
Why does $d = \min(d_X, d_Z)$ hold for a CSS code?
Any non-trivial logical $P = X^a Z^b$ has at least one non-trivial part ($a$ or $b$). Its weight $|\operatorname{supp}(a) \cup \operatorname{supp}(b)| \geq \max(|a|, |b|)$, which is $\geq d_X$ or $\geq d_Z$. So every non-trivial logical has weight $\geq \min(d_X, d_Z)$.
Recalled?

Conclusion

The proof rested on a single key idea: defining invariants $h$ and $v$ that detect whether a cycle wraps around the torus horizontally or vertically. These invariants are well-defined on homology classes, completely classify the four cosets of $H_1$, and force any non-trivial cycle to contain at least $L$ edges, because the regular structure of the square lattice guarantees $L$ disjoint "slices" each contributing at least one edge.

The deeper lesson is that the distance of the toric code is a topological fact, not a combinatorial coincidence. The number of logical qubits comes from $\dim(H_1)$, and the distance comes from the minimum weight of a non-trivial homology class. This is why the proof naturally lives in the language of homology.

This perspective generalizes far beyond the square lattice. The entire framework — qubits on $k$-cells, stabilizers from boundary maps, logical operators from non-trivial homology classes — works for any cellulation of any manifold in any dimension. The distance question becomes a question about the systole: the shortest non-trivial cycle. This is the foundation for hyperbolic codes, higher-dimensional toric codes, and the broader family of homological codes.

The planar and rotated surface codes have different boundary conditions that change the homology (from $H_1 \cong \mathbb{F}_2^2$ to $H_1 \cong \mathbb{F}_2$), and the distance argument must account for paths that terminate at boundaries rather than wrapping around.
